# Contributing

1. Open an issue before making a large behavioral change.
2. Use Java 17 or newer and keep the project free of Spring Boot and databases.
3. Run `mvn clean test` and `mvn clean package` before submitting a change.
4. Add focused tests for changed behavior.
5. Do not contribute copied code without a compatible license and preserved
   notices.
6. Keep commits focused and document user-visible changes in `CHANGELOG.md`.
